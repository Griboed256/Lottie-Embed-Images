### Инструкция по использованию
Приложение на версии v1.2.0 работает только на ОС Windows (проверялась на win11) и не поддерживает MacOS.

### Что делает приложение
Приложение преобразует Lottie-анимацию (экспорт из After Effects через Bodymovin) в один готовый JSON-файл так называемый self-contained. На выходе получается файл вида: *название_файла*_result.json 
В нём уже встроены все изображения в формате base64, поэтому папка images/ больше не потребуется.

### Быстрый сценарий работы
* Открыть приложение
* Выбрать в строке исходный .json экспортируемый из bodymovin в After Effects
* Указать папку images/ (если не подставилась автоматически)
* Выберети степень сжатия или оставте рекомендуемое (обычно это WebP и 85% качества)
* Нажать «Запустить»
* После завершения будет создан файл: **`*название файла*_result.json`**
* А так же будет открыт предпросмотр получившегося результата
* Только этот новый **`.json`** и отправляем разработчику

### Рекомендации по подготовке в After Effects
* Композиция: размер = финальному (например 270×270, 180х180, 90х90), 30 FPS, желательно до 300 кадров
* PNG-секвенция: импорт как Image Sequence, размер кадров совпадает с композицией, слой без масштабирования
* Если PNG больше композиции — его нужно уменьшить в AE вручную.
* Иначе JSON получится значительно тяжелее, так как он будет вставлять исходный размер изображения и уменьшать его кодом.

### Тоесть сценарий такой:
1) вы сформировали готовую анимацию объекта на карте. (не важно, вектор или растр это)
2) создали новую композицию нужного размера, например 270х270 px
3) отрендерили такого размера отдельно секвенцию через PNG Image Sequence
4) загрузили её ещё раз в after effects как PNG секвенцию
5) В этой новой композиции с готовой секвенцией проверили чтобы в названии файла не было .PNG в конце слоя (просто сотрите его)
6) Открываете Bodymovie и экспортируете как **`.json`** файл (не делайте сжатия и не встраивайте ничего в сам json) На выходе у вас должен получиться исходный json файл и папка image с секвенцией рядом с ним. 
После экспорта из Bodymovin у нас в папке должен быть исходный json и папка с секвенциями /image. как это будет выглядеть в “проводнике”:
*название проекта*/
 ├── *найзвание файла*.json
 └── images/
      ├── img_0001.png
      ├── img_0002.png
      └── ...
7)  Работаем с приложением
Запустите LottieEmbedImages.exe

### Перетащите Исходный JSON в приложение
Файл анимации, экспортированный из After Effects.
Можно: выбрать через кнопку “Обзор…” или перетащить файл в окно приложения 
Папка images, Папка с PNG-кадрами. Обычно определяется автоматически, если лежит рядом с JSON.

### Выберете формат сжатия
* WebP — оптимальный вариант по размеру и качеству
* WebP Lossless — без потерь, но тяжелее
* PNG — без изменений

### Вы так же можете ограничить размера JSON
Если включить этот параметр: формат и ручное качество отключаются приложение само подбирает степень сжатия итоговый файл старается уложиться в заданный лимит
НО! качествуо картинки может быть заметно ухудшенно, в подсказке смотрите на сколько уменьшается параметр сжатия 

### Оценка размера
Перед запуском показывается примерный итоговый размер файла.
Он рассчитывается по части кадров и может немного отличаться от финального результата.

### Запуск обработки
* После нажатия «Запустить»:
* отображается прогресс
* показывается оставшееся время
* выводится лог обработки
* После завершения:
* открывается предпросмотр в браузере
* открывается папка с результатом

### Результат
* Файл создаётся рядом из исходного JSON:
* example.json → example_result.json
* Что передавать разработчикам
* Только один файл: **`*название файла*_result.json`**
* Никакие дополнительные папки не нужны.

### Итог
Приложение: собирает анимацию в один файл оптимизирует размер убирает зависимость от images даёт быстрый предпросмотр. Без ручной сборки и дополнительных шагов.


### Частые ситуации
Файл получается слишком большим — включить ограничение размера или уменьшить качество
Заметная потеря качества — увеличь лимит размера или отключи ограничение
